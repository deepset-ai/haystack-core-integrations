9**Haystack Description** 

10An Open-Source Python framework for developers worldwide. 

11AI orchestration framework to build customizable, production-ready LLM applications. 

12Connect components (models, vector DBs, file converters) to pipelines or agents that can interact with your data. 

13With advanced retrieval methods, it's best suited for building RAG, question answering, semantic search or 

14conversational agent chatbots. 

15 

16**High-Level Architecture** 

17Haystack has two central abstractions: 

18- Components 

19- Pipelines 

20 

21A Component is a lightweight abstraction that gets inputs, performs an action and returns outputs. 

22Some example components: 

23- `OpenAIGenerator`: receives a prompt and generates replies to the prompt by calling an OpenAI-model 

24- `MetadataRouter`: routes documents to configurable outputs based on their metadata 

25- `BM25Retriever`: retrieves documents from a 'DocumentStore' based on the 'query'-input 

26 

27A component is lightweight. It is easy to implement custom components. Here is some information from the docs: 

28 

29Requirements 

30 

31Here are the requirements for all custom components: 

32 

33- `@component`: This decorator marks a class as a component, allowing it to be used in a pipeline. 

34- `run()`: This is a required method in every component. It accepts input arguments and returns a `dict`. The inputs can 

35either come from the pipeline when it's executed, or from the output of another component when connected using 

36`connect()`. The `run()` method should be compatible with the input/output definitions declared for the component. 

37See an [Extended Example](#extended-example) below to check how it works. 

38 

39## Inputs and Outputs 

40 

41Next, define the inputs and outputs for your component. 

42 

43### Inputs 

44 

45You can choose between three input options: 

46 

47- `set_input_type`: This method defines or updates a single input socket for a component instance. It's ideal for adding 

48or modifying a specific input at runtime without affecting others. Use this when you need to dynamically set or modify 

49a single input based on specific conditions. 

50- `set_input_types`: This method allows you to define multiple input sockets at once, replacing any existing inputs. 

51It's useful when you know all the inputs the component will need and want to configure them in bulk. Use this when you 

52want to define multiple inputs during initialization. 

53- Declaring arguments directly in the `run()` method. Use this method when the component's inputs are static and known 

54at the time of class definition. 

55 

56### Outputs 

57 

58You can choose between two output options: 

59 

60- `@component.output_types`: This decorator defines the output types and names at the time of class definition. The 

61output names and types must match the `dict` returned by the `run()` method. Use this when the output types are static 

62and known in advance. This decorator is cleaner and more readable for static components. 

63- `set_output_types`: This method defines or updates multiple output sockets for a component instance at runtime. 

64It's useful when you need flexibility in configuring outputs dynamically. Use this when the output types need to be set 

65at runtime for greater flexibility.

72from haystack import component

75class WelcomeTextGenerator:

77 A component generating personal welcome message and making it upper case 

78 ''' 

79 @component.output_types(welcome_text=str, note=str)

83```

91**Pipelines** 

92The pipelines in Haystack 2.0 are directed multigraphs of different Haystack components and integrations. 

93They give you the freedom to connect these components in various ways. This means that the 

94pipeline doesn't need to be a continuous stream of information. With the flexibility of Haystack pipelines, 

95you can have simultaneous flows, standalone components, loops, and other types of connections. 

96 

97# Steps to Create a Pipeline Explained 

98 

99Once all your components are created and ready to be combined in a pipeline, there are four steps to make it work: 

100 

1011. Create the pipeline with `Pipeline()`. 

102 This creates the Pipeline object. 

1032. Add components to the pipeline, one by one, with `.add_component(name, component)`. 

104 This just adds components to the pipeline without connecting them yet. It's especially useful for loops as it allows 

105 the smooth connection of the components in the next step because they all already exist in the pipeline. 

1063. Connect components with `.connect("producer_component.output_name", "consumer_component.input_name")`. 

107 At this step, you explicitly connect one of the outputs of a component to one of the inputs of the next component. 

108 This is also when the pipeline validates the connection without running the components. It makes the validation fast. 

1094. Run the pipeline with `.run({"component_1": {"mandatory_inputs": value}})`. 

110 Finally, you run the Pipeline by specifying the first component in the pipeline and passing its mandatory inputs. 

111 

112 Optionally, you can pass inputs to other components, for example: 

113 `.run({"component_1": {"mandatory_inputs": value}, "component_2": {"inputs": value}})`. 

114 

115The full pipeline [example](/docs/creating-pipelines#example) in [Creating Pipelines](/docs/creating-pipelines) shows 

116how all the elements come together to create a working RAG pipeline. 

117 

118Once you create your pipeline, you can [visualize it in a graph](/docs/drawing-pipeline-graphs) to understand how the 

119components are connected and make sure that's how you want them. You can use Mermaid graphs to do that. 

120 

121# Validation 

122 

123Validation happens when you connect pipeline components with `.connect()`, but before running the components to make it 

124faster. The pipeline validates that: 

125 

126- The components exist in the pipeline. 

127- The components' outputs and inputs match and are explicitly indicated. For example, if a component produces two 

128outputs, when connecting it to another component, you must indicate which output connects to which input. 

129- The components' types match. 

130- For input types other than `Variadic`, checks if the input is already occupied by another connection. 

131 

132All of these checks produce detailed errors to help you quickly fix any issues identified. 

133 

134# Serialization 

135 

136Thanks to serialization, you can save and then load your pipelines. Serialization is converting a Haystack pipeline 

137into a format you can store on disk or send over the wire. It's particularly useful for: 

138 

139- Editing, storing, and sharing pipelines. 

140- Modifying existing pipelines in a format different than Python. 

141 

142Haystack pipelines delegate the serialization to its components, so serializing a pipeline simply means serializing 

143each component in the pipeline one after the other, along with their connections. The pipeline is serialized into a 

144dictionary format, which acts as an intermediate format that you can then convert into the final format you want. 

145 

146> 📘 Serialization formats 

147> 

148> Haystack 2.0 only supports YAML format at this time. We'll be rolling out more formats gradually. 

149 

150For serialization to be possible, components must support conversion from and to Python dictionaries. All Haystack 

151components have two methods that make them serializable: `from_dict` and `to_dict`. The `Pipeline` class, in turn, has 

152its own `from_dict` and `to_dict` methods that take care of serializing components and connections.
