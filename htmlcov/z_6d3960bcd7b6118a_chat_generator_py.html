20class MistralChatGenerator(OpenAIChatGenerator): 

21 """ 

22 Enables text generation using Mistral AI generative models. 

23 

24 For supported models, see [Mistral AI docs](https://docs.mistral.ai/getting-started/models). 

25 

26 Users can pass any text generation parameters valid for the Mistral Chat Completion API 

27 directly to this component via the `generation_kwargs` parameter in `__init__` or the `generation_kwargs` 

28 parameter in `run` method. 

29 

30 Key Features and Compatibility: 

31 - **Primary Compatibility**: Designed to work seamlessly with the Mistral API Chat Completion endpoint. 

32 - **Streaming Support**: Supports streaming responses from the Mistral API Chat Completion endpoint. 

33 - **Customizability**: Supports all parameters supported by the Mistral API Chat Completion endpoint. 

34 

35 This component uses the ChatMessage format for structuring both input and output, 

36 ensuring coherent and contextually relevant responses in chat-based text generation scenarios. 

37 Details on the ChatMessage format can be found in the 

38 [Haystack docs](https://docs.haystack.deepset.ai/docs/data-classes#chatmessage) 

39 

40 For more details on the parameters supported by the Mistral API, refer to the 

41 [Mistral API Docs](https://docs.mistral.ai/api/). 

42 

43 Usage example: 

44 ```python 

45 from haystack_integrations.components.generators.mistral import MistralChatGenerator 

46 from haystack.dataclasses import ChatMessage 

47 

48 messages = [ChatMessage.from_user("What's Natural Language Processing?")] 

49 

50 client = MistralChatGenerator() 

51 response = client.run(messages) 

52 print(response) 

53

55 >> "Natural Language Processing (NLP) is a branch of artificial intelligence

57 >> meaningful and useful.")], _name=None,

60 ```

127 """ 

128 Creates an instance of MistralChatGenerator. 

129 

130 Unless specified otherwise in the `model`, this is for Mistral's `mistral-small-latest` model. 

131 

132 :param api_key: 

133 The Mistral API key. 

134 :param model: 

135 The name of the Mistral chat completion model to use. 

136 :param streaming_callback: 

137 A callback function that is called when a new token is received from the stream. 

138 The callback function accepts StreamingChunk as an argument. 

139 :param api_base_url: 

140 The Mistral API Base url. 

141 For more details, see Mistral [docs](https://docs.mistral.ai/api/). 

142 :param generation_kwargs: 

143 Other parameters to use for the model. These parameters are all sent directly to 

144 the Mistral endpoint. See [Mistral API docs](https://docs.mistral.ai/api/) for more details. 

145 Some of the supported parameters: 

146 - `max_tokens`: The maximum number of tokens the output text can have. 

147 - `temperature`: What sampling temperature to use. Higher values mean the model will take more risks. 

148 Try 0.9 for more creative applications and 0 (argmax sampling) for ones with a well-defined answer. 

149 - `top_p`: An alternative to sampling with temperature, called nucleus sampling, where the model 

150 considers the results of the tokens with top_p probability mass. So 0.1 means only the tokens 

151 comprising the top 10% probability mass are considered. 

152 - `stream`: Whether to stream back partial progress. If set, tokens will be sent as data-only server-sent 

153 events as they become available, with the stream terminated by a data: [DONE] message. 

154 - `safe_prompt`: Whether to inject a safety prompt before all conversations. 

155 - `random_seed`: The seed to use for random sampling. 

156 - `response_format`: A JSON schema or a Pydantic model that enforces the structure of the model's response. 

157 If provided, the output will always be validated against this 

158 format (unless the model returns a tool call). 

159 For details, see the [OpenAI Structured Outputs documentation](https://platform.openai.com/docs/guides/structured-outputs). 

160 Notes: 

161 - For structured outputs with streaming, 

162 the `response_format` must be a JSON schema and not a Pydantic model. 

163 :param tools: 

164 A list of Tool and/or Toolset objects, or a single Toolset for which the model can prepare calls. 

165 Each tool should have a unique name. 

166 :param timeout: 

167 The timeout for the Mistral API call. If not set, it defaults to either the `OPENAI_TIMEOUT` 

168 environment variable, or 30 seconds. 

169 :param max_retries: 

170 Maximum number of retries to contact OpenAI after an internal error. 

171 If not set, it defaults to either the `OPENAI_MAX_RETRIES` environment variable, or set to 5. 

172 :param http_client_kwargs: 

173 A dictionary of keyword arguments to configure a custom `httpx.Client`or `httpx.AsyncClient`. 

174 For more information, see the [HTTPX documentation](https://www.python-httpx.org/api/#client). 

175 """
