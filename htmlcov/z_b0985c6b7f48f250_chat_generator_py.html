14@component 

15class OrcaRouterChatGenerator(OpenAIChatGenerator): 

16 """ 

17 Enables text generation using OrcaRouter generative models. 

18 

19 OrcaRouter is an OpenAI-compatible model routing gateway that exposes 100+ chat models from providers such as 

20 OpenAI, Anthropic, Google, DeepSeek, and Qwen behind a single endpoint and API key. Models are addressed with a 

21 `provider/model` namespace (for example `openai/gpt-4o-mini` or `anthropic/claude-opus-4.8`). The special 

22 `orcarouter/auto` router selects an upstream model per request according to the routing policy configured in your 

23 OrcaRouter console. 

24 

25 For the list of supported models, see the [OrcaRouter model catalog](https://www.orcarouter.ai/models). 

26 

27 This component supports streaming, tool-calling, and structured outputs. 

28 It uses the ChatMessage format for both input and output; see the 

29 [Haystack docs](https://docs.haystack.deepset.ai/docs/chatmessage) for details. 

30 

31 Usage example: 

32 ```python 

33 from haystack_integrations.components.generators.orcarouter import OrcaRouterChatGenerator 

34 from haystack.dataclasses import ChatMessage 

35 

36 messages = [ChatMessage.from_user("What's Natural Language Processing?")] 

37 

38 client = OrcaRouterChatGenerator(model="openai/gpt-4o-mini") 

39 response = client.run(messages) 

40 print(response) 

41 ``` 

42 """

59 """ 

60 Creates an instance of OrcaRouterChatGenerator. 

61 

62 Unless specified otherwise, the default model is `openai/gpt-4o-mini`. 

63 

64 :param api_key: 

65 The OrcaRouter API key. 

66 :param model: 

67 The name of the OrcaRouter chat completion model to use. Models use a `provider/model` namespace 

68 (for example `openai/gpt-4o-mini`). Use `orcarouter/auto` to let OrcaRouter route the request. 

69 :param streaming_callback: 

70 A callback function that is called when a new token is received from the stream. 

71 The callback function accepts StreamingChunk as an argument. 

72 :param api_base_url: 

73 The OrcaRouter API base URL. For more details, see the OrcaRouter 

74 [documentation](https://docs.orcarouter.ai). 

75 :param organization: 

76 Your OrcaRouter organization ID, if any. 

77 :param generation_kwargs: 

78 Other parameters to use for the model. These parameters are sent directly to the OrcaRouter endpoint. 

79 See OrcaRouter [API docs](https://docs.orcarouter.ai) for more details. Some of the supported parameters: 

80 - `max_tokens`: The maximum number of tokens the output text can have. 

81 - `temperature`: The sampling temperature to use. Higher values mean the model takes more risks. 

82 - `top_p`: The nucleus sampling value to use. 

83 - `stream`: Whether to stream back partial progress. 

84 - `extra_body`: A dictionary of OrcaRouter-specific routing preferences (such as a fallback list of 

85 models) that is passed straight through to the gateway. 

86 :param tools: 

87 A list of tools or a Toolset for which the model can prepare calls. This parameter can accept either a 

88 list of `Tool` objects or a `Toolset` instance. 

89 :param tools_strict: 

90 Whether to enable strict schema adherence for tool calls. If set to `True`, the model follows exactly 

91 the schema provided in the `parameters` field of the tool definition. 

92 :param timeout: 

93 The timeout for the OrcaRouter API call. 

94 :param max_retries: 

95 Maximum number of retries to contact OrcaRouter after an internal error. 

96 If not set, it defaults to either the `OPENAI_MAX_RETRIES` environment variable, or set to 5. 

97 :param http_client_kwargs: 

98 A dictionary of keyword arguments to configure a custom `httpx.Client`or `httpx.AsyncClient`. 

99 For more information, see the [HTTPX documentation](https://www.python-httpx.org/api/#client). 

100 """
